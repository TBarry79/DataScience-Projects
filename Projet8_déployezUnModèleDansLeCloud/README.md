# Déployez un modèle dans le cloud

## Compétences Mises en Œuvre
- **Optimisation de l'infrastructure cloud** pour un environnement Big Data.
- **Mise en place d'une architecture scalable** pour le traitement d'images.
- **Parallélisation des opérations de calcul** avec PySpark.

## Description du Projet
En tant que Data Scientist chez "Fruits!", une start-up AgriTech, la mission est de développer une application mobile qui permet aux utilisateurs de prendre en photo un fruit et d'obtenir instantanément des informations à son sujet. La première étape consiste à créer une infrastructure Big Data sur AWS pour gérer et traiter les images de fruits à grande échelle.

## Source des Données
Le projet utilise un jeu de données d'images de fruits, accompagné des labels correspondants. Vous pouvez [Télécharger le jeu de données ici](https://www.kaggle.com/datasets/moltean/fruits)

## Objectif
Compléter la chaîne de traitement des données en utilisant PySpark dans un environnement AWS EMR. Cela implique de poser les fondations d'une infrastructure capable de gérer efficacement le traitement à grande échelle des images de fruits, tout en étant scalable et en respectant les contraintes réglementaires (RGPD)

## Contraintes

- Gérer une augmentation rapide du volume de données.
- Utiliser les services cloud AWS (EMR, S3, IAM) ou Databricks pour le traitement Big Data.
- Démontrer la fonctionnalité d'une instance EMR opérationnelle.
- Compléter le script PySpark en intégrant un traitement de diffusion des poids TensorFlow et une réduction de dimension avec PCA.
- S'assurer que les données sont traitées sur des serveurs situés en Europe pour respecter le RGPD.
- Optimiser les coûts en ne maintenant l'instance EMR opérationnelle que pour les tests et les démonstrations.
